# Career-Ops Dashboard v2

Static single-page dashboard — 4 actionable metrics from `applications.tsv`.

No build step. No dependencies. Vanilla JS + SVG + HTML tables.

## Usage

### Option A — Local HTTP server (recommended, works in all browsers)

From the **project root**:

```bash
npx serve .
# or: python3 -m http.server 8080
# or: VS Code → right-click index.html → Open with Live Server
```

Then open: `http://localhost:8080/dashboard-v2/`

### Option B — Open directly in Firefox

Firefox allows `file://` fetches within the same local tree.  
Double-click `dashboard-v2/index.html` — it just works.

> **Chrome / Edge:** block `file://` cross-origin fetches by default.  
> Use Option A, or launch Chrome with `--allow-file-access-from-files`.

## Data sources

The dashboard reads TSV files generated by `merge-tracker.mjs`:

| Profile | Path |
|---------|------|
| recepcionista | `profiles/recepcionista/data/applications.tsv` |
| formacion | `profiles/formacion/data/applications.tsv` |

Generate (or refresh) a TSV:

```bash
node merge-tracker.mjs --profile recepcionista
node merge-tracker.mjs --profile formacion
```

Then click **↻ Refresh data** in the dashboard.

## Metrics

| # | Metric | Description |
|---|--------|-------------|
| 1 | Conversion Funnel | Entries per pipeline stage: Scan → Evaluated → Applied → Responded → Interview → Offer |
| 2 | Response Rate by Channel | % of sent applications that got a reply, grouped by `channel` field |
| 3 | Unanswered Age | Days since applied, color-coded: green < 7d · yellow 7–14d · red > 14d |
| 4 | Score by Channel / Source | Avg, min, max score per `channel + source` combination |

Metrics 2 and 4 require schema v2 entries (channel/source fields, date ≥ 2026-06-23).  
Metrics 1 and 3 work with legacy 9-column entries too.
